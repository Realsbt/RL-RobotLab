# ToGo_LFs 后空翻复现与训练记录

更新日期：2026-07-20
状态：进行中

## 目标

在 Isaac Lab / RL-RobotLab 中，使用 ToGo_LFs 模型和普通 PPO 学得一次完整的原地后空翻。策略需要完成起跳、空中向后旋转一周、足端着地并恢复稳定支撑。训练、评估、失败现象、奖励修改和 checkpoint 选择都持续记录在本文档中。

本任务只针对仿真。未经关节顺序、力矩/转速、冲击、自碰撞和安全吊架验证，不部署到真实机器狗。

## 固定约束

- 机器人：`ToGo_LFs`，12 个关节。
- 算法：普通 PPO；不使用 CTS 或现有 CTS checkpoint。
- Actor 输入：60 维；critic 输入：72 维；动作：12 维关节位置残差。
- 控制频率：50 Hz；物理频率：200 Hz。
- 自碰撞：关闭。当前 ToGo 碰撞网格在开启全局自碰撞后会产生虚假的躯干/腿部接触。
- 首先追求名义条件下完整动作，再增加执行器延迟和窄范围随机化。

## 验收判据

单回合成功必须同时满足：

1. 曾经由至少两个支撑足进入腾空，且根节点至少上升 0.03 m、起跳瞬间竖直速度不低于 0.25 m/s。
2. 累积向后旋转角处于 `1.75π–2.25π`。
3. 完整旋转后发生可信着地，至少三个足端接触。
4. 着地后机身竖直方向偏差不超过 20°。
5. 稳定判定时线速度不高于 0.75 m/s、角速度不高于 2.0 rad/s。

最终名义策略的验收门槛：128 个并行环境、至少两个完整回合的确定性评估中，`landing_success >= 80%`。同时保存可视化回放指令，以人工确认动作确实是起跳后完成一周后翻，而不是擦地、躯干支撑或指标误判。

## 训练路线

1. 复核已有 jump/rotate/early-rotate checkpoint，使用统一的确定性评估选择真实最优旋转策略。
2. 若完整旋转不足，继续阶段 2/2A：强化支撑期向后角动量、上升段旋转和一周完成率，同时保留跳跃高度。
3. 完整旋转达到门槛后进入 landing 阶段：降低固定时间/姿态轨迹约束，保留一周旋转，学习收腿、展开和吸收冲击。
4. 名义着地成功后进入 robust 阶段，恢复 5–20 ms 延迟和窄范围动力学随机化。
5. 每轮只根据评估指标和回放暴露的失败模式修改奖励，不以训练 mean reward 单独判断成功。

### R2：从完整旋转策略直接学习稀疏落地（失败，已停止）

- 起始 checkpoint：R1 的 `model_1598.pt`。
- Task：`RobotLab-ToGo-LFs-Backflip-v0`。
- 第一轮预算：4096 环境，计划 600 PPO iterations；按失败判据在约 165 轮停止。
- 转移方式：只加载 actor/actor observation normalization；critic 重新初始化；optimizer 和 iteration 不恢复；初始动作噪声标准差 `0.10`。
- 目的：保持 `rotation_success` 的同时学习完整旋转后的三/四足接触、展开、低滑移和低速度稳定支撑。
- 保留 checkpoint：`logs/rsl_rl/togo_lfs_backflip_land_ppo/2026-07-17_20-14-45_land_from_rot1598_r2_600/model_150.pt`。
- 训练末期：起跳率 100%，历史最大角口径的旋转率约 81%，`landing_success = 0%`；稳定奖励持续为零。
- `model_150.pt` 确定性复评（128 环境、262 回合）：
  - 起跳率 100%，历史最大角旋转率 89.31%，躯干终止 6.20%。
  - 严格落地成功率 0%。
  - 首次触地时平均只有 1 个足接触，线速度 `2.959 m/s`，竖直速度 `-2.956 m/s`，角速度 `2.814 rad/s`。
  - 首次触地直立余弦 `-0.957`，即接近完全倒置。
- 回放观察：策略在空中一度达到接近一周的历史最大累计角，随后反向回转，约 0.88 s 倒置触地，再弹跳翻滚。
- 根因：
  1. 原 `rotation_success` 与落地角度奖励使用历史最大旋转，无法识别“曾到一周、触地前又退回半周”。
  2. 原稳定姿态误差只使用 projected gravity 的 x/y 分量，直立与完全倒置都会得到零姿态误差。
  3. 稳定项要求已经三足接触且低速，失败策略没有连续的接近梯度。
- 结论：单纯延长 R2 不会跨过落地奖励鸿沟，停止并修改奖励。

### R3：当前有符号角 + 连续落地接近塑形（失败，已停止）

- 起始 checkpoint：R2 `model_150.pt`，保留 actor，重置 critic/optimizer。
- 代码修改：
  - 着地成功的角度范围改用当前有符号累计旋转，不再使用历史最大值。
  - 落地旋转范围、触地角准确度、下降接近和伸腿塑形统一使用当前有符号旋转。
  - 姿态误差使用完整目标重力向量 `[0, 0, -1]`，明确区分直立和倒置。
  - 新增下降末段的直立/一周角度/俯仰刹车连续奖励。
  - 新增足端低于机身的伸腿准备奖励。
  - 接触后稳定奖励不再硬门控三足状态，而是按接触足比例、直立程度和线/角速度连续变化；严格成功判据不变。
  - 新增触地时有符号角、直立余弦、足数、线速度、竖直速度和角速度诊断。
- 修改后基线诊断：R2 `model_150.pt` 首次触地当前有符号旋转仅 `2.910 rad`，而历史最大角为 `6.050 rad`，验证了指标漏洞。
- 验证：任务纯逻辑测试 22/22 通过，Python 编译检查通过，2 环境 100 步仿真 smoke test 通过。
- 实际运行：4096 环境，计划 400 iterations，在约 63 轮主动停止；保留了 `model_0.pt`、`model_25.pt` 和 `model_50.pt`。
- 训练现象：首次触地有符号旋转从约 `2.9 rad` 进一步下降到约 `2.3 rad`，严格落地旋转率仅约 1%–2%，落地成功率仍为 0%。连续接近奖励在尚无完整空中旋转时形成了“较早停止旋转”的局部最优。
- 更深层根因：tracker 只在达到 `1.5π` 后才把触地记作 landed；不足角度的首次触地后，旋转仍持续累计，旧 `max_backward_rotation`、milestone 和 `rotation_success` 因而奖励了地面弹跳/翻滚。
- 结论：拒绝 R3 checkpoint。必须退回旋转阶段，首先严格定义空中旋转边界，再学习落地。

### R4：首次触地硬边界下的严格空中旋转（失败，已停止）

- 代码修正：
  - 首次任一足端触地是空中动作的硬终点；之后冻结有符号角和历史最大角。
  - `has_landed` 只能在首次触地事件上建立，不能在后续弹跳中补记。
  - Rotate、Early-Rotate 和 Land 阶段新增 `incomplete_touchdown`：首次触地角小于 `1.75π` 时立即终止。
  - 旋转进度、高度、轨迹和落地准备奖励均只在首次触地前激活。
  - 三个相关阶段都把 `incomplete_touchdown` 纳入终止失败惩罚。
- 验证：任务纯逻辑测试 23/23 通过；Python 编译检查通过；2 环境严格终止 smoke test 通过。
- 严格确定性复评（128 环境，每个 checkpoint 256 个回合）：
  - `long_consolidate/model_1598.pt`：首次触地角约 `3.210 rad`，最高点旋转 `0.791 rad`，严格旋转率 0%。
  - `early_apex/model_100.pt`：首次触地角约 `3.018 rad`，最高点旋转 `0.861 rad`，严格旋转率 0%。
  - `early_cav_consolidate/model_200.pt`：首次触地角约 `3.088 rad`，最高点旋转 `0.834 rad`，严格旋转率 0%。
- 选择：以严格首次触地角最高的 `long_consolidate/model_1598.pt` 为起点。
- 第一轮配置：`RobotLab-ToGo-LFs-Backflip-Rotate-v0`，4096 环境，普通 PPO，actor-only 转移并重置 critic/optimizer，初始动作噪声 `0.30`，计划 300 iterations。
- 实际运行：在 iteration 137 主动停止；checkpoint 为 `logs/rsl_rl/togo_lfs_backflip_rotate_ppo/2026-07-17_20-35-48_strict_aerial_r4_300/model_100.pt`。
- 训练结果：严格旋转率始终为 0%，触地前旋转由初始约 `3.21 rad` 波动并下降到约 `3.0 rad`；起跳率仍为 100%，峰值高度约 `0.75 m`。
- 诊断：支撑期 `launch_centroidal_pitch_rate` 回报只有约 `0.2–0.3`，而腾空后的 CAV 回报约 `33`；起跳瞬间后翻俯仰速度仍为负值。策略继续依赖腾空后的腿/机身交换形成半周姿态，没有学会从地面反力获得足够后翻角动量。
- 结论：拒绝 R4 checkpoint，不再单独延长本奖励组合。切换到已公开验证的 Genesis 端到端奖励和 PPO 超参数完成完整预算。

### R5：Genesis 端到端 LFS 完整预算（失败，已完成）

- 依据：本地 `external/Genesis-backflip` 上游默认使用 10000 环境、1000 PPO iterations、`learning_rate=1e-3`、`entropy_coef=0.01`、动作噪声 `1.0`，奖励直接在 `0.5–1.0 s` 鼓励后翻俯仰速度，并同时塑形起跳高度和完整姿态轨迹。
- 本项目的 `RobotLab-ToGo-LFs-Backflip-Genesis-v0` 已独立复现这些核心设置，但机器人、关节约定、48 Nm 力矩-速度包络和碰撞模型均保持 ToGo_LFs；因此仍是 LFS 上的普通 PPO。
- 起点：R0 `model_100.pt`，恢复 actor、critic、optimizer 和 iteration；R0 未保存的 101–177 轮不计入。
- 计划：4096 环境继续 900 iterations，即从 iteration 100 训练到约 999；先完整执行上游默认轮数，再用首次触地硬边界指标确定性验收。
- 完成 checkpoint：`logs/rsl_rl/togo_lfs_backflip_genesis_ppo/2026-07-17_20-40-40_genesis_lfs_full_r5/model_999.pt`。
- 训练末期：起跳率 100%，峰值高度约 `0.608 m`，起跳瞬间向后俯仰速度约 `9.70 rad/s`，但首次触地前最大角仅约 `3.35 rad`，严格旋转率和落地率均为 0%。
- 现象：约 iteration 358 时触地角一度达到 `3.24 rad`，之后 mean reward 持续提高到约 `18.23`，但触地角长期停留在半周附近；原 `pitch_velocity` 在首次触地后仍持续到 1.0 s，允许策略靠地面继续俯仰提高回报。
- 结论：在 ToGo_LFs 的动力学和动作约定上，原 Genesis 奖励按默认 1000 轮收敛到半周触地/地面翻滚局部最优；完整训练预算本身不是缺失原因。保留 R5 作为反例，不进入落地阶段。

### R6：Genesis-Strict 名义空中角度课程（事件奖励缺陷，已停止）

- 新 Task：`RobotLab-ToGo-LFs-Backflip-Genesis-Strict-v0`。
- 与 R5 的区别：
  - 使用 ToGo_LFs 零动作延迟、无域随机化的名义资产，先获取动作再做鲁棒化。
  - Genesis 后翻俯仰速度奖励只在首次触地前有效。
  - 空中旋转进度由首次触地硬边界 tracker 给出，地面翻滚不再计分。
  - 新增首次触地事件的单步角度质量奖励，质量为 `(触地前角度 / 2π)^2`；只发一次，避免早触地后反复累计。
  - 保留起跳高度、支撑期后翻俯仰、姿态轨迹和末段站立高度塑形。
- 算法：普通 PPO，沿用 Genesis 的高探索配置（初始噪声 `1.0`、学习率 `1e-3`、entropy `0.01`）。
- 验证：新增任务和奖励通过 24/24 纯逻辑测试及 Python 编译检查；启动训练前再执行 2 环境仿真 smoke test。
- 实际运行：4096 环境，从零训练到 iteration 145；保留 `logs/rsl_rl/togo_lfs_backflip_genesis_strict_ppo/2026-07-17_20-53-08_strict_nominal_r6_1000/model_100.pt`。
- 结果：100% 起跳，峰值约 `0.592 m`，空中历史最大角约 `3.58 rad`，首次触地当前角约 `3.30 rad`；严格完整旋转率仍接近 0%。
- 新诊断：策略会先达到约 `1.81 rad` 再反向，证明触地奖励必须使用触地时当前有符号角而非历史最大角；同时事件奖励在日志中始终为 0，说明 0.5 个 control step 的检测窗口窄于 manager 的更新时序。
- 修正：触地质量改用 `signed_backward_rotation_at_touchdown`，事件窗口扩到 1.5 个 control steps（仍只覆盖接触事件附近，不能长期累计）。
- 下一步：从 `model_100.pt` 只迁移 actor，重置 critic/optimizer，以修正后的触地回报继续 R6b。

### R6b：真实触地角单步质量（发现非足端接地漏洞，已停止）

- 起点：R6 `model_100.pt`，actor-only，探索噪声 `0.8`。
- Task：`RobotLab-ToGo-LFs-Backflip-Genesis-Strict-v0`，4096 环境。
- 目的：保留已获得的跳跃/旋转，再直接优化首次触地的单调向后角度，消除先转后反转。
- 运行中在 iteration 330 达到训练分布下约 `4.74 rad` 触地角和 0.34% 严格旋转，但约 iteration 423 发生 PPO 灾难性更新，起跳率骤降；在 iteration 436 停止。
- 崩溃前候选：`model_300.pt`；`model_350.pt` 次之，`model_400.pt` 已退化。
- 确定性复评暴露新漏洞：`model_300.pt` 在两秒内没有足端触地记录，却仍累计 `4.655 rad`。这意味着策略可能先以躯干/腿部接地，而 tracker 仍把它当作腾空。
- 严格口径再升级：首次足端或任意非足端刚体接地都冻结空中旋转；非足端首次接地单独记录，只有没有非足端同时碰撞的足端首次接触才可建立 `has_landed`。
- 升级后 `model_300.pt` 确定性复评：0.78 s 首次接地、非足端接地率 100%、足端数 0、触地累计角 `4.581 rad`、机身直立余弦却为 `0.962`。后两者几何不一致，说明积分世界坐标 y 角速度仍可被 roll/yaw 三维进动利用。
- 旋转轴再修正：累计角改为机身局部 `root_ang_vel_b[:, 1]`，即机器狗自身左右轴；配合现有 yaw、roll/重力方向惩罚，避免把三维进动误判为后空翻。
- 机身局部角速度复评仍得到约 `4.608 rad`，与触地直立余弦 `0.962` 不相容。因此最终严格旋转定义改为 projected gravity 在机身 x-z 平面的解包绕转角：真正后翻必须形成直立→倒置→直立的完整重力向量绕转；x-z 范数小于 0.5（roll 过大）时该步不累计。

#### R10g 结果与 R10h 结构化末段动作

- R10g 运行目录：`logs/rsl_rl/togo_lfs_backflip_genesis_land_ppo/2026-07-20_15-33-40_conservative_terminal_front_r10g_800`，4096 环境，seed 63，在 iteration 122 主动停止。尽管学习率已降到 `5e-6`，训练分布的完整旋转率仍先下降到约 `80%`，FL/FR x 仅到约 `-0.130/-0.126 m`；奖励微调无法隔离共享网络的起跳与落地段，拒绝 checkpoint。

- 直接用 R9i `model_225.pt` 做 256 回合确定性评估：完整旋转保持 `100%`，触地角 `5.694 rad`；FL/FR x 从无辅助的 `-0.1686/-0.1682 m` 改善到 `-0.0893/-0.0877 m`，最长干净支撑从 `0.203 s` 增至 `0.674 s`。结构路线首次同时改善前伸和支撑时长，但前脚仍未跨过机身原点。
- R10h 保持相同角度窗口，在默认前腿站姿目标上加入对称过冲：FL hip-pitch/knee 偏置 `-0.40/-0.40 rad`，FR 为 `+0.40/+0.40 rad`。目标是在不更早展开、不过多损失转角的前提下，用更大的末段位置误差加快前脚摆动；先直接确定性评估，再决定是否训练 PPO 补偿。

#### R10h 参数扫描与 R10i PPO 补偿失败

- 结构化动作层首先证明了前腿方向诊断正确：把介入窗口提前到 `1.45π–1.70π` 后，R9i `model_225.pt` 的 FL/FR 首触 x 从约 `-0.168 m` 推到机身原点附近，但总转角下降到约 `5.43 rad`。触地后继续保持过冲目标会削弱支撑，因此改为落地后返回承重姿态。
- 对介入角度和髋/膝目标做 256 环境确定性扫描。过早介入可以前伸但欠转；过晚介入保持整圈但前伸不足。URDF 正向运动学也验证了“髋和膝同时过冲”会把足端向上收，不能只按静态 x 位置选择目标。
- R10i 运行目录：`logs/rsl_rl/togo_lfs_backflip_genesis_land_ppo/2026-07-20_15-52-30_terminal_front_compensate_r10i_400`，4096 环境，seed 64，从 R9i `model_225.pt` 只迁移 actor，普通 PPO、动作噪声 `0.10`、学习率 `5e-6`；在约 iteration 125 主动停止。
- R10i 的离地角速度由约 `13.12` 增至 `14.18 rad/s`，但离地前累计转角同时由约 `1.49` 降至 `1.28 rad`，总触地转角反而由约 `5.41` 降到 `5.36 rad`；前脚也逐渐回退。拒绝全部 R10i checkpoint，继续训练不会自动解决共享 actor 的时序耦合。

#### R10j：两段式末段前摆与四腿平滑承重（严格通过）

- 最终仍使用普通 PPO 的 R9i `model_225.pt` 作为 actor，不采用退化的 R10 checkpoint。Task 固定为 `RobotLab-ToGo-LFs-Backflip-Genesis-Land-Assisted-v0`。
- 空中段：下降且后转角从 `1.52π` 到 `1.75π` 时，将 FL/FR 目标平滑混合到对称前摆姿态；FL hip-pitch/knee 偏置为 `-0.45/-0.40 rad`，FR 为 `+0.45/+0.40 rad`。该窗口在 50 Hz 控制离散化下保留完整旋转安全余量。
- 触地段：首次干净足触地时不瞬间跳到站姿，而是先保持当前前摆/屈腿目标吸收冲击，再用 `0.30 s` 将全部 12 个关节目标平滑混合到 LFS 默认站姿。实验表明瞬时四腿锁定只产生约 `0.252 s` 干净支撑；平滑过渡才会稳定站立。
- Assisted episode 延长为 `3.0 s`，只为给触地后的低速稳定窗口留下验收时间；翻转和首次触地仍在原时序内完成（首次触地约 `0.74 s`），空中 policy/动力学不变。
- 成功判据补严：稳定窗口除了至少三足、直立、线速度 ≤`0.75 m/s`、角速度 ≤`2 rad/s` 外，必须持续无任一非足端碰地，避免趴地静止的假成功。
- 最终 256 环境、无观测噪声、150 步确定性评估：起跳率、完整旋转率、有效旋转轴、干净首次触地率、严格落地成功率均为 `100%`；总触地转角 `5.6165 rad`，触地直立余弦 `0.9969`，首次触地 2 足、后续最大 4 足。
- FL/FR 首触 x 为 `-0.0667/-0.0564 m`，相比原模型的约 `-0.168 m` 已前移约 10–11 cm；随后到达 `+0.1570/+0.1572 m`。最终诊断时 RL/RR 为 `-0.0974/-0.1010 m`，支撑区已跨过机身质心。
- 最长连续干净三足以上支撑为 `0.8245 s`，同时满足无非足接触、直立与低速门槛的连续稳定时间为 `0.5891 s`，超过 `0.20 s` 严格门槛。R10j 因此作为当前名义 LFS 后空翻的完成版本。

#### R11：后空翻 PPO 与 CTS locomotion 拼接

- 新增 `deploy/deploy_mujoco/deploy_togo_lfs_backflip_locomotion.py`，后空翻继续使用 R10j 的 R9i actor 和末段结构化动作层，locomotion 使用已有 45 维单帧输入、内部 10 帧历史的 CTS TorchScript policy。两个 policy 不合并网络权重，由有限状态机分时控制。
- 启动时保持标准站立姿态，手柄 `A` 或 MuJoCo 窗口内 `Space` 触发后空翻。翻身期间速度指令强制为零。
- 交接不依赖历史上已锁存的 `landing_success`，而是要求当前时刻至少三足、无非足碰地、机身直立、低线/角速度且关节接近站姿，连续保持 `0.30 s`。
- 交接时先调用 CTS `reset()`，再用当前状态和零指令预热 10 帧历史；随后在 `0.40 s` 内用 smoothstep 对绝对关节目标做平滑混合，并继续保持零指令 `0.50 s`，才允许手柄输入。
- locomotion 期间再次按触发键会先进入准备状态：清零速度指令，等待更严格的静止站立门槛持续 `0.30 s`，再重置后空翻策略和时钟。
- 名义 MuJoCo 无界面端到端验收通过：后空翻最大旋转 `1.79π`，最终直立度 `0.999`，四足支撑、无非足接触，CTS 接管后零指令连续运行 `2.0 s` 仍通过当前稳定性判定。
- 本阶段验证的是 togo-master 所用同款 XML 的 MuJoCo sim2sim，尚未改动 togo-master 的真机 C++ 控制器；真机移植时仍需复制相同状态机、历史重置和安全门槛。

#### R12：togo-master 原生 C++/ONNX/FSM 拼接

- 改为直接集成 `<togo-master>` 工作区，不再把独立 Python MuJoCo runner 作为主播放入口。保留原生 `S_Idle -> T_EnableMotors -> T_StifflyStand -> S_StiffStanding -> S_PolicyLocomotion` 坐姿起立流程。
- 新增 `S_PolicyBackflip` 状态和 `/policy/backflip` ONNX 模型。复制后的 `backflip.onnx` 与 R9i 导出模型 SHA-256 均为 `c804b55ea4b41732aaf4552b3d1cb09553115c61abde7369d8f99b66cde03615`。
- 手柄新建独占组合键 `LB + A`，回调只在 `S_PolicyLocomotion` 中接受。触发后先把 locomotion 命令历史锁为零，等待直立、低线/角速度且关节接近站姿连续 `0.30 s`，才转入后空翻。其他状态下 `LB + A` 无效。
- C++ 状态原样构造 60 维观测：机身角速度、投影重力、相对关节位置、关节速度、当前/上一动作和 6 维相位特征；保留 `0.5` 动作尺度、50 Hz policy 频率和 R10j 末段前腿前摆/触地后 `0.30 s` 站姿混合。
- 首次原生端到端运行虽然能自动回 locomotion，但触地旋转达 `2.365π`，属于明显过转。原因是 togo-master mjsim 原本只有恒定 `±48 Nm` 限幅，没有训练/已验收播放器使用的 268 rpm 电机扭矩—转速包络。
- 在 `S_PolicyBackflip` 的 500 Hz 力矩下发中加入同款四象限扭矩—转速包络，并将 MuJoCo 资产对齐为用户已选的无自碰撞、滑动摩擦 `0.75`。
- 最终原生无界面全链路验收：坐姿起立后进入 CTS，转入后空翻后 `0.240 s` 检测起跳，`0.720 s` 检测触地，触地旋转 `1.817π`；落地连续稳定 `0.30 s` 后自动返回 `S_PolicyLocomotion`，再保持零指令 `0.50 s` 后解锁摇杆。
- 为回归验收添加了仅 simulation build 生效的 `TOGO_SIM_AUTORUN_BACKFLIP=1` 环境变量路径；正常 `run_sim.bash` 不设置该变量，仍必须由真实手柄完成使能、切换 locomotion 和 `LB + A` 触发。
